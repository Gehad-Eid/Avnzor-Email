2 Survey response (thank you + answers)

About

Send after someone submits a survey: thank-you line, subline, then two question/answer rows. In these static files, replace the sample {{QUESTION_*}} and {{ANSWER_*}} tags in the HTML with real content (or your tool outputs merged HTML). For a single merge file for EN and AR, use avnzor-feedback-response-email-CRM.html in the section below. Field reference: field guide, section 5.

How to use

  1. Map survey fields into the placeholders, or paste text for a one-off test.
  2. Optional emoji in ANSWER_*_EMOJI; short label in ANSWER_*_TEXT.
  3. Test in Gmail and Outlook before automating.

Survey design guidelines (for the form you build)

Questions should be short and scannable (ideally one line, under ~100 characters), clear, and in plain language. The email will show the full question text in bold on the left, so avoid long paragraphs.

How answers look in the email (right side): one emoji plus a short label on the same line, e.g.:

Example (stacked, as in the template) ☺️  Satisfied 😔  Disappointed

1 to 5 scale (recommended for satisfaction-style questions): in your survey tool, use a 1-5 choice. When you build the message to customers, map the numeric answer to an emoji and label the template expects:

Score Suggested emoji (for ANSWER_*_EMOJI) Suggested label (for ANSWER_*_TEXT)
1 😔 Very disappointed
2 😕 Disappointed
3 😐 Neutral
4 🙂 Satisfied
5 ☺️ or 😊 Very satisfied

Your team can change emoji or labels to match brand tone, as long as each response still maps to one emoji string and one label string per question row. For non-scale questions, still use a short label (and an emoji if it helps, or leave the emoji field empty).

4 Survey invite (merge template)

File

avnzor-feedback-email-CRM.html

About

Same design as the static invite; all visible copy, lang / dir, and the survey CTA are merge fields. One file can serve English and Arabic if your tool fills the tokens per audience. Set {{CTA_URL}} to the live form link. See the field guide for the full list.

How to use

  1. Map each {{placeholder}} to your platform merge field (or replace once for a test).
  2. Align {{HTML_LANG}} / {{HTML_DIR}} (and body, wrapper) with the language of the strings you inject.
  3. Test send to internal inboxes before going live.
